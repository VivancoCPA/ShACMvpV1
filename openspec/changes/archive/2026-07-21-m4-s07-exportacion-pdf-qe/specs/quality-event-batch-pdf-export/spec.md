## ADDED Requirements

### Requirement: Batch export produces one PDF per selected QE, packaged into a single zip
When the user clicks "Exportar seleccionados" in `QEList`'s toolbar with N selected QEs (1 ≤ N ≤ 50), the system SHALL, for each selected QE sequentially: invoke the `useExportQualityEventPdf` mutation (same endpoint used by the individual export, registering its own `EXPORTACION_PDF` audit trail entry), then call `buildQualityEventPdf` with the mutation's response QE and `meta = { exportadoPorNombre: <current user's full name>, generadoEn: new Date() }`, convert the resulting document to a `Blob` via `doc.output('blob')`, and add it to a `JSZip` instance under the filename `${qe.numero}.pdf`. Once all N QEs are processed, the system SHALL generate the zip via `zip.generateAsync({ type: 'blob' })` and trigger a single browser download named `quality-events-export-<YYYYMMDD-HHmm>.zip`, using the project's established Blob-download pattern (`<a download>` + `URL.createObjectURL`/`revokeObjectURL`).

#### Scenario: Batch export of 3 QEs produces a zip with 3 correctly-named PDFs
- **WHEN** a user selects `QE-2026-001`, `QE-2026-002`, and `QE-2026-003` in `QEList` and clicks "Exportar seleccionados"
- **THEN** exactly one `.zip` file downloads, containing `QE-2026-001.pdf`, `QE-2026-002.pdf`, and `QE-2026-003.pdf`

#### Scenario: Each PDF within the batch is generated by the same function as an individual export (RN-QE-019)
- **WHEN** `QE-2026-001` is exported individually from `QualityEventDetail`, and separately as part of a 3-QE batch export
- **THEN** both invocations call `buildQualityEventPdf(qe, meta)` with the same `QualityEvent` content, producing byte-for-byte identical section content (differing only in the `meta.generadoEn` timestamp)

#### Scenario: PDFs are generated sequentially, not concurrently
- **WHEN** a batch export of 5 QEs is triggered
- **THEN** the `useExportQualityEventPdf` mutation for QE #2 is not dispatched until QE #1's mutation has resolved and its PDF has been added to the zip

### Requirement: Batch export is capped at 50 QEs (RN-QE-020)
`QEList`'s "Exportar seleccionados" button SHALL be disabled whenever the current selection count exceeds 50, and SHALL display a message stating the 50-QE maximum. The button SHALL remain enabled for any selection count from 1 to 50 inclusive.

#### Scenario: Selecting 51 QEs disables the export button with a message
- **WHEN** a user selects 51 QEs in `QEList` (e.g. via "Seleccionar todos los visibles" on a filtered view with 51 results)
- **THEN** "Exportar seleccionados" is disabled and a message indicating the 50-QE limit is shown

#### Scenario: Selecting exactly 50 QEs keeps the export button enabled
- **WHEN** a user selects exactly 50 QEs in `QEList`
- **THEN** "Exportar seleccionados" remains enabled

#### Scenario: Reducing selection back to 50 or fewer re-enables the button
- **WHEN** a user's selection goes from 51 QEs down to 40 QEs
- **THEN** "Exportar seleccionados" becomes enabled again and the limit message is no longer shown
